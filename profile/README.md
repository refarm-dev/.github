## 🌍 Our Mission: Digital Citizenship & Data Sovereignty

**Refarm** is more than a software project; it is a movement toward a **Sovereign Web**. We are building the foundations for a virtual hackerspace dedicated to digital citizenship, open-source education, and taking back control of our digital lives from centralized SaaS monopolies.

We believe your data should live on your device, under your control, and be portable anywhere, without being locked into proprietary clouds.

## 🏗️ The Sovereign Ecosystem

Refarm is a unified architecture that manifests as distinct experiences. All domains run the same core microkernel but curate different capabilities:

- 💻 **`refarm.dev`**: The core engine, SDKs, and developer portal.
- 🧠 **`refarm.me`**: The sovereign identity and private "Second Brain" interface.
- 🌐 **`refarm.social`**: The public network, federated communities, and P2P gardens.

### 🛠️ Core Technology Stack
We build on the "bleeding edge" of web infrastructure:
- **Local-First Architecture:** OPFS (Origin Private File System) + Browser-native SQLite.
- **Universal Plugins:** WASM (WebAssembly) host for isolated, language-agnostic plugins.
- **Conflict-Free Sync:** Advanced CRDTs (Conflict-free Replicated Data Types).
- **Agnostic Identity:** Pluggable identity layers with native support for decentralized protocols like **Nostr**.
- **UI Agnostic:** Web Components & JSON-LD standard.

## 🤝 Join the Hackerspace (How to Contribute)

We are actively seeking researchers, developers, and architects passionate about the Local-First paradigm and WebAssembly.

1. **Understand the Architecture:** Read our[ADRs (Architecture Decision Records)](https://github.com/refarm-dev/refarm/tree/main/specs/ADRs) and [Research Validations](https://github.com/refarm-dev/refarm/tree/main/docs/research).
2. **Our Rigorous Workflow:** We strictly follow a `SDD → BDD → TDD → DDD` pipeline. See our [Workflow Guide](https://github.com/refarm-dev/refarm/blob/main/docs/WORKFLOW.md).
3. **Get Involved:** Check our[CONTRIBUTING.md](https://github.com/refarm-dev/refarm/blob/main/CONTRIBUTING.md) and look for issues tagged with `good first issue` or `help wanted`.
